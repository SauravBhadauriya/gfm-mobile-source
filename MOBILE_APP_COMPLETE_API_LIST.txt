==========================================
DATA FLOW: APP ↔ ADMIN PANEL
==========================================

1. COMPETITIONS:
   Admin creates → POST /admin/competitions
   App fetches → GET /competitions (public, no auth)
   App joins → POST /competitions/{id}/join
   App uploads entry → POST /competitions/{id}/entries
   Admin views participants → GET /admin/competitions/{id}/participants
   Admin declares winners → POST /admin/competitions/{id}/declare-winners
   Backend auto-creates earnings → User wallet updated
   App shows earnings → GET /user/earnings

2. EARNINGS:
   User wins competition → Backend auto-creates earnings
   User receives tip → POST /user/{userId}/tip → Backend creates earnings
   App shows earnings → GET /user/earnings
   Admin views earnings → GET /admin/earnings

3. USER DETAILS:
   User updates profile → PUT /user/profile
   Backend saves → User data updated
   Admin views user → GET /admin/users/{id}

4. KYC:
   User completes steps → PUT /user/profile (with bio, profileImage, etc.)
   Frontend auto-verifies → Checks all steps complete
   Backend updates status → KYC status = 'completed'
   Admin views KYC → GET /admin/users/{id}/kyc

5. REELS:
   User uploads reel → POST /reels/upload
   Backend AI/ML moderation → Auto-approve/reject
   Approved reels visible → GET /reels
   Admin views reels → GET /admin/reels (if needed)

6. REPORTS:
   User reports → POST /reports
   Backend creates report → Visible in admin
   Admin views → GET /admin/reports
   Admin resolves → PUT /admin/reports/{id}/status

7. WITHDRAWALS:
   User requests → POST /user/wallet/withdraw
   Backend creates request → Status: 'PENDING'
   Admin processes → (Manual process in admin)
   User views history → GET /user/wallet/withdraw-history

SPECIAL NOTES
==========================================

1. CATEGORIES API:
   - Currently uses /admin/categories (requires auth)
   - MUST change to /categories (public, no auth)
   - Backend needs to make this endpoint public

2. COMPETITION FLOW:
   - Admin creates competition → Status: CREATED/UPCOMING
   - Admin approves → Status: LIVE
   - App fetches LIVE competitions → Shows in app
   - User joins → POST /competitions/{id}/join
   - User uploads entry → POST /competitions/{id}/entries
   - Admin declares winners → Coins auto-distributed
   - Earnings auto-created → Visible in app and admin

3. REEL MODERATION:
   - User uploads → POST /reels/upload
   - Backend AI/ML processes → Auto-approve/reject
   - Approved reels → Status: APPROVED, visible in app
   - Rejected reels → Status: REJECTED, not visible
   - No admin intervention needed

4. EARNINGS AUTO-UPDATE:
   - Competition win → Backend auto-creates earnings
   - Tip received → Backend auto-creates earnings
   - Both visible in app (GET /user/earnings) and admin (GET /admin/earnings)

5. KYC AUTO-VERIFICATION:
   - User completes all steps → Frontend checks
   - Frontend calls PUT /user/profile → Backend saves
   - Frontend triggers auto-verify → Backend updates KYC status
   - Admin can view → GET /admin/users/{id}/kyc

6. CHAT FEATURES:
   - Send message → POST /chat/{userId}/messages
   - Delete message → DELETE /chat/messages/{messageId}
   - Forward message → POST /chat/messages/{messageId}/forward
   - Archive conversation → PUT /chat/conversations/{userId}/archive
